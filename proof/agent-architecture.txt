│  SwarmOS · Agent Architecture · Three Reasoning Paradigms                    │
╰──────────────────────────────────────────────────────────────────────────────╯

  Paradigm 1: Sync Fan-out (delegate_task)
    Pattern:  spawn N subagents in parallel → collect results → synthesize
    Use case: independent tasks (recon + deploy simultaneously)
    Overhead: ~3s setup, parallel execution, single-turn

  Paradigm 2: Async Chain (cron context_from)
    Pattern:  Job A collects → Job B processes → Job C delivers
    Use case: pipelines (data → analysis → notification)
    Overhead: scheduled, cross-job context injection

  Paradigm 3: Recursive Single-turn
    Pattern:  one LLM turn, 10+ sequential tool calls
    Use case: complex inline tasks (WL screening, form submission)
    Overhead: ~2m40s, ~17k tokens, no subagent spawn
